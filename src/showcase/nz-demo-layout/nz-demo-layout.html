Layout

Assists with overall page-level layout.

Design Rules

size

A navigation left side of the logo placed near the left side of the auxiliary menu to place.

  • Top navigation (most systems): Level 1 navigation 64px Level 2 navigation 48px

  • Top navigation (display category page): first level navigation 80px second level navigation 56px

  • The range of top navigation heights is calculated as: 48 + 8n

  • The range of side navigation widths is calculated as: 200 + 8n

Interact

  • Primary and final navigation needs to be emphasized at the visual level;

  • The current item should be rendered with the highest priority;

  • When navigation is collapsed, the style of the current item is automatically given to its previous level;

  • The retractable interaction in the left navigation bar supports both accordion and fully-expanded styles, making the appropriate selection based on your business requirements.

Visual

Navigation style requires a reasonable selection style based on the level of information:

  • Highlighting

    It is recommended for navigation items of the parent of the current page when the underlying color is dark.

  • Highlight a matchstick

    It is used when the background color of the navigation bar is a light color. It can be used for navigation items corresponding to the current page. It is suggested to use it as the last item in the navigation path.

  • Font highlighting discoloration

    From a visual perspective, fonts are highlighted with less visual enhancement than large patches, often at the top of the current item.

  • Enlarge Font

    12px 14px is the standard font size for navigation, font 14 is used in one or two levels of navigation. Size can be considered the level of navigation items to make the appropriate choice.

Component Overview

  • nz-layout Layout container, under which nested nz-header nz-sider nz-content nz-footer or nz-layout itself, can be placed in any parent container.

  • nz-header The top layout, comes with a default style, under which any elements can be nested, only in nz-layout .

  • The nz-sider sidebar comes with its default styles and basic features, which can be nested beneath any element, only in nz-layout .

  • The content section of the nz-content comes with a default style, where any elements can be nested, only in nz-layout .

  • nz-footer The bottom layout, comes with a default style, under which any elements can be nested, only in nz-layout .

Note: The use of flex layout, please note Browser Compatibility problem.

Typical page layout.

The most basic "upper-middle-lower" layout.

The main navigation placed at the top of the page, from left to right are: logo, a navigation item, auxiliary menu (user, settings, notifications, etc.). Usually the content is placed in a fixed size (for example: 1200px), the entire page layout stability, not affected by the display on the user terminal; the upper and lower structure in line with the user's browsing habits, but also the more classic site navigation mode. The way the page is divided up and down improves the efficiency of information display in the main work area, but there are some sacrifices in the vertical space. In addition, due to the limitation of the horizontal space of the navigation bar, it is not suitable for the information structure with a large number of level one navigation items.

Also has the top navigation and sidebar, the difference is not margins on both sides, and more for application-based website.

The page with the top navigation and sidebar is mostly used to showcase categories of websites.

Side two-column layout. Side navigation is limited when the page has limited horizontal space.

The navigation on the side of the page uses left and right structure. The main navigation is usually placed on the left side of the page and the auxiliary menu is placed on the top of the workspace. The content is adaptive to the browser terminal, increasing the utilization of landscape space, but the entire page layout is not stable. Side navigation model hierarchy scalability, one, two, three navigation items can be more smoothly and related to be displayed, while the side navigation can be fixed, allowing users to navigate and navigate in the fast positioning and switching The current location, there is a high operating efficiency. However, such navigation horizontal page content space will be sacrificed.
