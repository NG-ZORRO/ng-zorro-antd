In most business cases, Ant Design needed to solve a large number of information storage problems in the design area, so based on the 12-grid system, we divided the entire design recommendation area into 24 equal parts.

Divided information blocks are what we call "boxes." It is recommended to arrange up to four boxes horizontally and at least one. "Box" in the entire screen as shown above. The design section is based on a box-based unit that customizes the typesetting rules inside the box to ensure visual comfort.

Overview

For a rasterized layout of a layout, we define the outer frame of the tile based on rows and cols to ensure that each area of ​​the page is well-organized. Here's a brief introduction to how it works:

  • Create a set of column (abbreviated cols) horizontally with row

  • Your content should be placed inside col, and only col can be a direct element of row

  • Columns in a raster system are values ​​from 1 to 24 representing the range over which they span. For example, three equally-wide columns can be created using .col-8

  • If the sum of col in a row exceeds 24, the extra col is put together as a whole

Flex Layout

Our rasterization system supports a Flex layout that allows horizontal alignment of child elements within parent nodes - left, center, right, width, and so on. Sub-elements and sub-elements, support for top alignment, vertical alignment, the bottom of the alignment. At the same time, support the use of order to define the order of the elements.

The Flex layout defines the width of each "box" based on the 24 grid, but the layout is not limited to the grid.
