Modal Dialog

modal dialog box.

When to use

When you need to manipulate a transaction without wanting to jump the page to interrupt the workflow, use Modal to open a float in the middle of the current page to host the action.

It is recommended to use the method of loading Component to pop up Modal, so that the component logic that can pop the layer can be completely isolated from the upper component and can be reused anytime

In the pop-up layer Component can be passed to the outer component of the NzModalSubject

Alternatively, use a well-encapsulated NzModalService.confirm () method when you need a clean confirmation box to ask the user.

NzModalSubject Object

In the pop-up component to instantiate NzModalSubject object through the next method to transfer data out

All NzModalService.xxx () methods return an NzModalSubject object.

This object is the wrapper for the Subject object of rxjs. In addition to supporting the native method of the Subject object, it also supports:

  • NzModalSubject.destroy (type) - Used to destroy the current modal object
  • NzModalSubject.on (eventType, cb) - Event for listening to various stages of a modal object

The type support in the destroy method passes in onOk and onCancel to indicate onCancel or onOk methods in the options passed by the user when destroying modal

All eventTypes supported by NzModalSubject include:

  • onShow

  • onShown

  • onHide

  • onHidden

  • onOk

  • onCancel

  • onDestroy

  • Other (users can customize any event that can be monitored by this object)
